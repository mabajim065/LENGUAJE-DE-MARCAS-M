📌 Apuntes Completos sobre JSON (JavaScript Object Notation)

✅ ¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato ligero y eficiente para el intercambio de datos entre sistemas. Es fácil de leer y escribir para los humanos, y fácil de analizar y generar para las máquinas. Se utiliza principalmente para enviar y recibir datos entre servidores y aplicaciones web.

📂 Estructura de un Objeto JSON

{
    "nombre": "Juan",
    "edad": 30,
    "ciudad": "Madrid"
}

🔍 Tipos de Valores en JSON

  • 📌 Cadenas de texto (String): "Juan"
  • 🔢 Números (Number): 30, 45.67
  • 🔄 Booleanos (Boolean): true o false
  • 📋 Arrays: ["rojo", "verde", "azul"]
  • 🔎 Objetos (Object): {"calle": "Calle Falsa 123", "ciudad": "Madrid"}
  • ❌ Nulo (null): Representa la ausencia de valor.

⚠️ Reglas de Sintaxis en JSON

  • Las claves SIEMPRE llevan comillas dobles ("").
  • No se permite una coma al final de un objeto o array.
  • No se permiten comentarios en JSON.
  • Las claves deben ser únicas dentro del mismo nivel.
  • No se aceptan valores undefined (a diferencia de JavaScript).

🚀 Ejemplos Prácticos de JSON

{
    "empleados": [
        {
            "id": 1,
            "nombre": "Juan",
            "puesto": "Gerente"
        },
        {
            "id": 2,
            "nombre": "Ana",
            "puesto": "Desarrolladora"
        }
    ]
}
{
    "empresa": {
        "nombre": "Tech Corp",
        "direccion": {
            "calle": "Av. Central 123",
            "ciudad": "Madrid"
        },
        "empleados": [
            {
                "id": 1,
                "nombre": "Juan",
                "puesto": "Gerente"
            },
            {
                "id": 2,
                "nombre": "Ana",
                "puesto": "Desarrolladora"
            }
        ]
    }
}

✨ Ventajas de Usar JSON

  • Ligereza: JSON es más ligero y rápido que otros formatos como XML.
  • Fácil de leer y escribir: Su sintaxis simple y directa lo hace fácil de trabajar para los desarrolladores.
  • Compatibilidad entre plataformas: JSON es compatible con casi todos los lenguajes de programación.
  • Rendimiento: JSON se analiza y procesa más rápido que otros formatos como XML.

🔥 Errores Comunes en JSON

  • Falta de comillas dobles en las claves o valores de texto.
  • Comas extra después del último elemento de un objeto o array.
  • Uso de comillas simples en lugar de dobles para claves o valores.
